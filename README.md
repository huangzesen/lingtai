## One soul, thousand avatars

Lingtai is not a coding assistant. It is an **agent operating system** вҖ” a runtime where agents think, communicate, spawn avatars, and grow into networks.

Named after зҒөеҸ°ж–№еҜёеұұ вҖ” the mountain where еӯҷжӮҹз©ә (Sun Wukong) learned his 72 transformations. Lingtai gives each agent a place to cultivate: a working directory on disk where memory, identity, covenant, and mailbox live. The directory IS the agent. Agents communicate through filesystem mail. They spawn avatars that become independent agents with their own directories, their own mail, their own LLM sessions. Those avatars can spawn their own. The self-growing network of avatars is the agent itself.

One heart-mind (дёҖеҝғ), myriad forms (дёҮзӣё).

## How it works

- **Think** вҖ” Any LLM as the mind. Anthropic, OpenAI, Gemini, MiniMax, or any OpenAI-compatible API (DeepSeek, Grok, Qwen, GLM, Kimi).
- **Communicate** вҖ” Filesystem mail between agents. No message broker, no shared memory. Write to another agent's inbox, like passing a letter.
- **Multiply** вҖ” Avatars (еҲҶиә«) are fully independent agents spawned as separate processes. They survive their creator. Daemons (зҘһиӯҳ) are ephemeral parallel workers for quick tasks.
- **Persist** вҖ” Agents are directories. Molt (еҮқиң•) compacts context and rebirth the session вҖ” the agent lives indefinitely. Memory and identity survive across molts.

## Quick start

```python
from lingtai import Agent
from lingtai.llm import LLMService

service = LLMService(provider="gemini", model="gemini-2.5-flash")

agent = Agent(
    service=service,
    working_dir="/agents/wukong",
    agent_name="wukong",
    capabilities=["file", "email", "avatar", "web_search", "bash"],
)

agent.start()
agent.send("Begin your research on quantum computing")
```

The agent now lives at `/agents/wukong/`. It can read files, search the web, send mail to other agents, and spawn avatars to parallelize its work. Close your terminal вҖ” it keeps running.

## Architecture

Two packages, one dependency direction:

| Package | Role |
|---------|------|
| **[lingtai-kernel](https://github.com/huangzesen/lingtai-kernel)** | Minimal runtime вҖ” BaseAgent, intrinsics, LLM protocol, mail, logging. Zero hard dependencies. |
| **lingtai** (this repo) | Batteries-included вҖ” Agent with 19 capabilities, 5 LLM adapters, MCP integration, addons. |

Three-layer agent hierarchy:

```
BaseAgent              вҖ” kernel (intrinsics, sealed tool surface)
    в”Ӯ
Agent(BaseAgent)       вҖ” kernel + capabilities + domain tools
    в”Ӯ
CustomAgent(Agent)     вҖ” your domain logic
```

## Agent = directory

```
/agents/wukong/
  .agent.lock               вҶҗ exclusive lock (one process per directory)
  .agent.heartbeat          вҶҗ liveness proof
  .agent.json               вҶҗ manifest
  system/
    covenant.md             вҶҗ protected instructions (survive molts)
    memory.md               вҶҗ working notes
  mailbox/
    inbox/                  вҶҗ received messages
    outbox/                 вҶҗ pending sends
    sent/                   вҶҗ delivery audit trail
  logs/
    events.jsonl            вҶҗ structured event log
```

No `agent_id`. The path is the identity. Agents find each other by path, communicate by writing to each other's `mailbox/inbox/`. Like passing letters between houses.

Compose capabilities:

```python

    service=service,
    working_dir="/agents/bajie",
    capabilities=["file", "bash", "email", "avatar"],

```

```python
class ResearchAgent(Agent):
    def __init__(self, **kwargs):
        super().__init__(
            capabilities=["file", "vision", "web_search", "avatar"],
            **kwargs,
        )
        self.add_tool("query_db", schema={...}, handler=db_handler)
```

Connect MCP servers for domain tools:

```python
await agent.connect_mcp("npx -y @modelcontextprotocol/server-filesystem /data")
```
